1. Интерфейс CalculatorOperation с единственным абстрактным методом perform()
  - класс Addition реализует интерфейс CalculatorOperation
  - класс Division реализует интерфейс CalculatorOperation

    Интерфейс CalculatorOperation закрыт для изменнеия, т.к его изменение может повлиять на реализации и привести к
    нежелательным последствиям. При этом можно добавлять новые операции, реализуя этот интерфейс или, применив наследование
    к уже существующим реализациям. В этом проявляется возможность расширения.

2. Класс Calculator имеет единственный метод calculate(CalculatorOperation operation). Этот класс так же закрыт для изменений,
    то есть реализацию метода менять нельзя, да и нет необходимости. В калькуляторе "появляются" новые операции за счет
    механизма раштрения, описанного в пункет 1.
    Таким образом класс Calculator тоже закрыт для изменений, но открыт для расширения за счет механизма наследования,
    т.к метод calculate принимает аргументом все объекты, являющиеся типом CalculatorOperation.
